Healthcheck

OpenClaw 팁: validate·health·cleanup으로 운영 기본기부터 자동화하기

OpenClaw를 한동안 굴려보면, 새 기능을 쫓는 것보다 더 중요해지는 순간이 옵니다. 바로 **“지금 이 인스턴스가 건강하게 돌아가고 있는가”**를 짧고 확실하게 확인하는 일입니다. 모델 추가, 채널 연동, 크론 자동화, 서브에이전트 운영도 물론 중요하지만, 실제로 오래 안정적으로 쓰려면 먼저 운영 기본기가 탄탄해야 합니다. 최근 OpenClaw 쪽에서 특히 실용적으로 느껴지는 패턴은 config validate, /healthz·/readyz, 그리고 세션 정리 루틴을 한 묶음으로 보는 방식입니다. 이 조합이 좋은 이유는 간단합니다. 많은 운영 문제는 화려한 장애가 아니라 애매한 불안정성으로 시작하기 때문입니다. 설정은 저장됐는데 실제로는 canonical 키가 어긋나 있거나, 게이트웨이는 살아 있는데 준비 상태가 덜 됐거나, 세션 저장소에 찌꺼기가 쌓여서 나중에 이상 동작이 생기는 식입니다. 이런 문제는 각각 따로 보면 사소해 보이지만, 시간이 지나면 “왜 오늘은 이상하지?” 같은 피로를 만들죠. 그래서 OpenClaw 운영에서는 작은 검증 3개를 습관처럼 묶는 게 꽤 효과적입니다. ...
