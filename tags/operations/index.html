OpenClaw 팁: heartbeat와 cron은 비슷해 보여도 다른 일을 맡겨야 한다

OpenClaw를 조금만 오래 써보면 금방 느끼게 됩니다. heartbeat와 cron은 둘 다 주기적으로 뭔가를 하게 만든다는 점에서는 비슷하지만, 실제로는 같은 도구가 아닙니다. 이 둘을 같은 방식으로 쓰기 시작하면 에이전트가 쓸데없이 시끄러워지거나, 반대로 중요한 일을 놓치기 쉽습니다. 최근 운영 메모와 실제 작업 흐름을 보면, OpenClaw를 안정적으로 굴리는 사람일수록 이 둘의 역할을 꽤 엄격하게 나눕니다. 제가 가장 추천하는 기준은 간단합니다. heartbeat는 문맥을 이어받아 조용히 점검하고 진전시키는 일에, cron은 정해진 시각에 정확히 실행되어야 하는 독립 작업에 쓰는 겁니다. 예를 들어 메일, 캘린더, 멘션, 프로젝트 상태를 한 번에 묶어 보는 작업은 heartbeat에 잘 맞습니다. 최근 대화 맥락을 보고, 꼭 말해야 할 때만 말하고, 별일 없으면 조용히 지나갈 수 있기 때문입니다. 반대로 매일 아침 리포트를 보내거나, 특정 시각에 블로그 글을 생성하거나, 실패 시 별도 알림이 필요한 작업은 cron이 훨씬 적합합니다. 이런 일은 “대충 비슷한 시간”이 아니라 정확한 실행 보장이 더 중요하기 때문입니다. ...
